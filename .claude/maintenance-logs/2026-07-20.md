## Task 2: Security Audit

Full re-read of `backend/server.py` (4245 lines) against all prior maintenance logs to avoid
re-flagging fixed issues.

**Fixed — 2 more moderation gaps, same class of bug as 2026-07-17's sweep:**
- `PUT /api/users/profile` checked `display_name`/`bio` via `is_content_flagged()` but not
  `weight_class` or `stance` — both echoed publicly through `safe_user()` on public profiles,

  `/search/fighters`. Extended to both fields.

- `POST /api/auth/session` (Emergent OAuth session exchange — creates accounts and sets the
  session cookie from a client-supplied `session_id`) had no rate limit, unlike `/auth/register`
  (5/60s) and `/auth/login` (10/60s). Added the same `_rate_limited` guard (10/60s by IP).

`python3 -m py_compile backend/server.py` passes.

**Checked, already correct — no change needed:** auth coverage on all 118 route handlers (public
endpoints are public by design: register/login/session, webhooks, static content, `/ads/*`,
`/waitlist/signup`); IDOR/ownership checks on every delete/update; token/tip economy uses atomic
`$inc` (no double-spend); Stripe amounts always server-side; no raw Mongo string interpolation
anywhere (`search_fighters` uses `re.escape`). Reviewed and consciously left as-is: `WaitlistSignup`'s

exploit path).

# Daily Maintenance — 2026-07-14

## Task 1: UI Micro-Improvements

**Touch targets + accessibility — icon-only close buttons**
`ShareSheet.jsx`, `PublicProfilePage.jsx` (followers/following modal), and `GymsPage.jsx` (both the
create-gym and join-gym modals) all had a bare `<X>` close button with no `aria-label` and no defined
touch target. Brought all four to the standard `w-11 h-11 touch-target` + `aria-label="Close"` pattern.

## Task 2: Security Audit

**Finding (fixed): unvalidated `video_url` accepted from the client in two endpoints.**
`register_uploaded_video` (`POST /api/videos/register`) stored a client-supplied `video_url` with no
validation it actually came from Cloudinary. More importantly, `analyze_video_with_vision`
(`POST /api/ai/analyze-video`) took a client-supplied `video_url` straight from the request body and
passed it to OpenAI's Vision API with no validation at all — a user could point it at any arbitrary
URL, burning paid AI quota analyzing non-boxing content (or worse, using Victory AI's OpenAI account
as a URL-fetch proxy). Added the same Cloudinary-prefix check already used for `avatar_url` in
`update_extended_profile` (`_AVATAR_PREFIXES`) to both endpoints.

Also spot-checked `generate_round_feedback` — takes structured JSON (`video_analysis`), not a URL, no
issue there.

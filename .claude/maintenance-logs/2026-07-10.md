## Task 2 · Security Audit

**No code changes needed.** Findings:

- ✅ CORS: scoped to specific Vercel origins, not wildcard
- ✅ Stripe webhook: refuses unsigned events when `STRIPE_WEBHOOK_SECRET` unset
- ✅ Rate limiting: register (5/min), login (10/min), tips (30/min), posts (10/min), comments (20/min)
- ✅ MongoDB search: uses `re.escape()` on user input
- ✅ JWT_SECRET: logs a warning if using the insecure default
- ✅ All sensitive endpoints use `Depends(get_current_user)`
- ⚠️ **Livepeer webhook (`/api/livepeer/webhook`):** Only verifies HMAC signature when `LIVEPEER_WEBHOOK_SECRET` is set. If the env var is absent from Railway Variables, unauthenticated callers can spoof stream status (e.g. mark a stream as live/idle). Stripe webhook correctly rejects in this case. Low priority since stream status is low-stakes, but worth setting the secret.

## Task 2: Security Audit

**Finding (fixed): `leave_gym` could kick a user out of the wrong gym.**
`DELETE /api/gyms/{gym_id}/leave` (`leave_gym` in `backend/server.py`) unconditionally cleared the
caller's `user.gym_id` field regardless of whether they were actually a member of the `gym_id` in the
URL. If a client called leave on a gym the user *wasn't* currently in (stale frontend state, or a
direct API call with an unrelated gym_id), it would silently detach them from their real gym while
leaving the target gym's membership untouched. Added a check that `user.gym_id` matches the requested
`gym_id` before doing anything, and only clear it if the member-pull actually modified a document.

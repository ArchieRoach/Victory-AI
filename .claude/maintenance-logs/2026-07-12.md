## Task 2: Security Audit

Checked all 115 `api_router` endpoints for missing auth, reviewed CORS config, scanned for hardcoded
secrets, and checked user-input-into-Mongo-query paths.

- **CORS**: explicit origin allowlist via `CORS_ORIGINS` env var (fallback list is Vercel domains +
  localhost), not a wildcard. Fine.
- **Hardcoded secrets**: none found — all key/token/secret reads go through `os.environ`.
- **Stripe webhook**: still refuses unsigned events (`STRIPE_WEBHOOK_SECRET` required, signature verified
  via `stripe_lib.Webhook.construct_event`). Fine.
- **Regex injection**: `search_fighters`'s `q` param is `re.escape()`d before use in `$regex`. Fine.
- **Auth coverage**: every endpoint touching user/account data requires `Depends(get_current_user)`.
  Public endpoints (`/dimensions`, `/drills`, `/legends`, `/plans`, `/health`, `/onboarding/*`,
  `/waitlist/signup`) are static reference data or pre-auth by design.
- **Finding (fixed): `/ads/checkout` had no rate limiting.** It's intentionally public (external
  advertisers, not app users — matches the public `AdvertisePage.jsx`), but unlike `/waitlist/signup`
  and `/auth/login`, it had no `_rate_limited` guard, so it could be spammed to create unlimited
  `pending_payment` ad_campaign documents and Stripe Checkout Sessions. Added the same
  `_rate_limited(f"ads_checkout:{client_ip}", 5, 60)` guard used on `/waitlist/signup`.

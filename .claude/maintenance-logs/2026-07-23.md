**Fixed — new `GET /users/me/export` had no rate limit:**
Added by the GDPR compliance commit to satisfy Art. 15/20 (data portability). It's correctly
behind `get_current_user`, but runs up to 15 unbounded (`to_list(10000)`) Mongo queries per
call with no throttle — unlike every other write-heavy or fan-out endpoint in the file, which
all go through `_rate_limited`. An authenticated user (or a compromised token) could hammer it
to load the DB. Added `_rate_limited(f"data_export:{user_id}", 3, 60)`, consistent with the
existing pattern.
